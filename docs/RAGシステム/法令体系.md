# RAGシステムへの入力文書：法令体系の整理

RAGシステムに取り込む文書は、法的効力の強弱に応じた**階層構造**を持つ。  
上位の文書ほど拘束力が強く、下位の文書は上位に反することができない。

---

## 階層図

```
┌──────────────────────────────────────┐
│  ① 法律                             │  ← 国会が制定。最上位の一般的拘束力
├──────────────────────────────────────┤
│  ② 政令                             │  ← 内閣が制定。法律の委任を受けた施行令等
├──────────────────────────────────────┤
│  ③ 府省令（府令・省令）             │  ← 内閣総理大臣令・各省大臣が制定
├──────────────────────────────────────┤
│  ④ 告示【法定型：省令委任】         │  ← 省令の委任に基づき市民にも直接効力
├──────────────────────────────────────┤
│  ⑤ 規則（国）                       │  ← 公正取引委員会・会計検査院等が制定
├──────────────────────────────────────┤
│  ⑥ 条例（県）                       │  ← 都道府県議会が制定。法令の範囲内
├──────────────────────────────────────┤
│  ⑦ 条例（市）                       │  ← 市区町村議会が制定。法令・県条例の範囲内
├──────────────────────────────────────┤
│  ⑧ 規則（地方）                     │  ← 首長が制定。条例に基づく執行規定
│    ★基準書【仕様書型】              │    工事契約に組み込まれ契約上の拘束力を持つ
├──────────────────────────────────────┤
│  ⑨ 告示（一般型・公示）             │  ← 省庁・自治体が広く周知するもの
├──────────────────────────────────────┤
│  ⑩ 通達・訓令                       │  ← 省庁の行政内部命令。市民への直接効力なし
│    ★基準書【通達型】               │    国の技術基準。補助金条件として実務上遵守
├──────────────────────────────────────┤
│  ⑪ 要項                            │  ← 制度・補助金等の申請条件・資格要件を規定
├──────────────────────────────────────┤
│  ⑫ 要領                            │  ← 事務・手続きの具体的な取り扱い方を規定
├──────────────────────────────────────┤
│  ⑬ ガイドライン                     │  ← 推奨・指針。法的拘束力なし
├──────────────────────────────────────┤
│  ⑭ 基準書【団体・業界基準型】       │  ← 学会・JIS・業界協会等。任意規格だが業界標準
├──────────────────────────────────────┤
│  ⑮ マニュアル                       │  ← 操作・作業手順書。実務の標準化
├──────────────────────────────────────┤
│  ⑯ 運用                            │  ← 現場レベルの取り決め・慣行
└──────────────────────────────────────┘
```

## 各区分の説明

| 優先順位 | 区分 | 制定主体 | 法的拘束力 | 土木分野での代表例 |
|---|---|---|---|---|
| 1 | **法律** | 国会 | 強（最上位） | 河川法、道路法、建築基準法、公共工事品質確保促進法 |
| 2 | **政令** | 内閣 | 強（法律に次ぐ） | 河川法施行令、道路法施行令、建設業法施行令 |
| 3 | **府省令**（府令・省令） | 内閣総理大臣・各省大臣 | 強（政令に次ぐ） | 河川法施行規則、建設業法施行規則、国土交通省令 |
| 4 | **告示【法定型】**（省令委任） | 各省庁（省令委任に基づく） | 強（府省令と同等） | 道路橋示方書（国交省告示）、建築基準法関係告示 |
| 5 | **規則（国）** | 国の行政委員会等 | 強（府省令に次ぐ） | 公正取引委員会規則、会計検査院規則 |
| 6 | **条例（県）** | 都道府県議会 | 中（域内に限定） | 〇〇県公共工事設計業務等の品質確保に関する条例 |
| 7 | **条例（市）** | 市区町村議会 | 中（域内に限定） | 〇〇市工事請負契約条例 |
| 8 | **規則（地方）** | 首長（知事・市長） | 中（条例の委任） | 工事施工規則、入札参加資格規則 |
| 9 | **告示（一般型）** | 省庁・自治体 | 弱（公示・周知） | 指定区域告示、事業認定告示、特定区間告示 |
| 10 | **通達・訓令** | 省庁（内部命令） | 行政内部のみ | 施工管理基準通達、設計業務等共通仕様書通達 |
| 11 | **要項** | 行政機関 | 弱（行政内部） | 工事発注事務処理要項、補助金交付要項 |
| 12 | **要領** | 行政機関 | 弱（行政内部） | 設計変更ガイドライン適用要領、積算要領 |
| 13 | **ガイドライン** | 行政機関 | 参考（拘束なし） | 設計変更ガイドライン、VE活用ガイドライン |
| 14 | **基準書【団体・業界基準型】** | 学会・業界団体・標準化機関 | 参考（任意規格・業界標準） | 土木学会基準、JIS規格、建築学会基準、日本道路協会技術基準、NEXCO設計要領 |
| 15 | **マニュアル** | 行政機関・発注者 | 参考（拘束なし） | 工事監督マニュアル、施工管理マニュアル |
| 16 | **運用** | 各担当部署 | 参考（慣行） | 事務処理手順書、担当者申し合わせ事項 |

## 基準書の位置づけ

「基準書」は単一の法的カテゴリではなく、**根拠・発行主体によって階層内の位置が変わる**。

| 基準書の種類 | 対応する階層 | 法的根拠・拘束力 | 代表例 |
|---|---|---|---|
| **告示型技術基準** | ④（府省令と同等） | 省令委任告示に基づく。設計に直接適用義務 | 道路橋示方書（国交省告示）、建築基準法関係告示 |
| **通達型技術基準** | ⑩通達・訓令に相当 | 省庁通達。行政内部のみ拘束。補助金条件として実務上遵守 | 河川砂防技術基準、道路土工要領 |
| **仕様書（契約組込型）** | ⑧規則（地方）に相当 | 工事請負契約書に組み込まれ、当事者間で法的効力を持つ | 土木工事共通仕様書、機械設備工事共通仕様書 |
| **団体・業界基準** | ⑭（独立区分） | 法的拘束力なし。業界標準・任意規格として参照。契約への組み込みで実務拘束力を持つ場合あり | 土木学会基準、JIS規格、建築学会基準、日本道路協会技術基準、NEXCO設計要領 |

> **RAG登録時の注意**: 同じ「基準書」という名称でも、告示型と通達型では矛盾解決時の優先度が大きく異なる。  
> メタデータの `doc_type` に上記の種類を明記し、根拠法令・告示番号を紐付けて登録すること。

## 階層図の妥当性検証とSSOTとしての適用

### 検証根拠

e-Gov 法令検索のへルプQ1より：  
> 「e-Govが提供する法令 = **憲法・法律・政令・勅令・府令・省令・規則**」  
> **告示・通達は法令に包まれず**（各府省等のHPで別活提供）。

### 正しい点

| 項目 | 評価 |
|---|---|
| ①法律 → ②政令 → ③府省令 の順位 | e-Govの正式分類（法律・政令・府省令・規則）と一致 |
| ④告示（法定型）を府省令の直下に配置 | 省令委任に基づく告示は府省令と同等の法的効力 |
| ⑤規則（国）を独立区分として配置 | e-Gov分類の「規則（国の行政委員会等）」に対応 |
| ⑥条例（県）→⑦条例（市）→⑧規則（地方） | 地方自治法の分類と一致 |
| ⑨告示（一般型）・⑩通達を法令体系外として明示 | e-Gov Q1「告示・通達は法令に含まれない」に対応 |
| ⑪要項〜⑯運用 | 実務的な位置づけとして妥当 |

### 修正した点（今回の変更）

| 内容 | 理由 |
|---|---|
| **②法令を②政令・③府省令に分割** | 政令（内閣制定）と省令（各省大臣制定）は制定主体・法的地位が異なるため個別管理が必要 |
| **告示を④（法定型）と⑨（一般型）の2種に分類し明示** | 省令委任の有無で法的効力が大きく異なる（委任あり→府省令と同等、なし→公示のみ） |
| **通達・訓令を⑩として独立した区分に明示** | e-Gov Q1「告示・通達は法令に含まれない」＝行政内部命令として明確化 |
| **⑤を「規則（国）」、⑧を「規則（地方）」と区別して明記** | e-Govの「規則（国の行政機関）」と地方自治法の「規則（首長）」を明確に区別 |

### SSOTとしての結論

**主要機能は妥当**。修正後の階層図は法令エージェントのSSOT優先順位として使用可能。  
ただし、**次の補完ルールをセットで運用**すること。

### SSOT補完ルール（同一階層内の競合解決）

上位階層の優先原則だけでは解決できない競合には、以下のルールを順に適用する。

| 優先順 | ルール | 内容 |
|---|---|---|
| 1 | **後法優先** | 同一階層内で新しい法令が古い法令に優先する |
| 2 | **特別法優先** | 同一階層内で特別法が一般法に優先する |
| 3 | **管轄原則** | 条例・規則は当該自治体の管轄内でのみ適用 |
| 4 | **通達型の実務適用範囲** | 通達型基準書は行政内部のみ拘束。市民・事業者決定に直接法的効力なし |

> **通達型基準書の実務上の扱い**: 法的には条例に優先しないが、国の補助金・奇付金の条件として遵守が求められる場合が多い。  
> RAGが回答生成する際は「法的必須」と「行政内部での実務上必須」を分けて根拠を提示すること。

## RAGシステムでの取り扱い方針

### 優先度付けの基本原則

矛盾が生じた場合、**上位区分の文書が下位区分に優先**する。  
AIが回答を生成する際は、必ず根拠とした文書の区分・名称・条項を明示する。

```yaml
# メタデータ例（文書登録時に付与）
doc_type: "法律"          # 上記①〜⑯の区分
doc_name: "河川法"
article: "第26条"
enacted_by: "国会"
effective_date: "1964-07-10"
jurisdiction: "全国"      # 全国 / 都道府県名 / 市区町村名
supersedes: []            # この文書が廃止・上書きする旧文書
```

### 区分別の登録・更新ルール

| 区分 | 更新頻度 | 確認先 | 備考 |
|---|---|---|---|
| ①法律 | 低（改正時のみ） | [e-Gov 法令検索](https://laws.e-gov.go.jp/registdb/) | 官報告示で確定 |
| ②政令 | 低〜中 | [e-Gov 法令検索](https://laws.e-gov.go.jp/registdb/) | 委任元の法律と紐付けて登録 |
| ③府省令 | 低〜中 | [e-Gov 法令検索](https://laws.e-gov.go.jp/registdb/) | 委任元の政令と紐付けて登録 |
| ④告示（法定型） | 低〜中 | 各省庁HP・官報 | 委任元の省令条項を必ず明記 |
| ⑤規則（国） | 低〜中 | 各委員会等HP | 所管委員会を明記 |
| ⑥⑦条例（県・市） | 中 | 各自治体例規集 | 自治体ごとに個別管理 |
| ⑧規則（地方） | 中 | 各自治体例規集 | 条例との紐付けを明記 |
| ⑨告示（一般型） | 中 | 各省庁・自治体HP | 官報番号を記録 |
| ⑩通達・訓令 | 高 | 各省庁HP | 発出年月日・文書番号を記録 |
| ⑪要項・⑫要領 | 高 | 各部署発出文書 | 版管理・有効期限に注意 |
| ⑬ガイドライン・⑭団体業界基準 | 中 | 国交省・学会・業界団体 | 非拘束だが最新版に更新 |
| ⑮マニュアル・⑯運用 | 高 | 各担当部署 | 現場慣行の変化を反映 |
